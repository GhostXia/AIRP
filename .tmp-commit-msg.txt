feat(revision): #115 Phase 2c-2g — remaining asset services 接入统一 revision 合同

在 Phase 2a（底层模块）+ Phase 2b（Preset）基础上，将剩余 5 个 asset service
接入统一 revision/provenance 合同。Phase 2h（trace 收口）单独一个 PR。

## Phase 2c: Character card
- 新增 `commit_character_revision` helper（`handlers/characters.rs`）
- `import_card_to_disk`：在 `extract_card_assets` 后 commit revision
- `update_character_card`：从裸 `fs::write` 改为 `replace_file` + `commit_revision`
- 批准文件：`card.json` + `raw.json`（greetings/lorebook 是派生产物，不纳入）
- lazy migration：首次 `content_revision=1`

## Phase 2d: Worldbook
- `LorebookService::write`：在 `replace_file` 写 `lorebook.json` 后 commit revision
- `asset_dir` = `characters/{id}/world`，`AssetKind::Worldbook`
- 批准文件：`lorebook.json`
- lazy migration

## Phase 2e: State
- `StateService::write`：在写 `live.json` + append `history.jsonl` 后 commit revision
- 直接复用 `snapshot.revision` 作为 `content_revision`（State 已有 revision 计数器）
- `asset_dir` = `characters/{id}/state`，`AssetKind::State`
- 批准文件：`state.json`（内容与 `live.json` 对齐，只含 state 字段）
- `source_kind` = "derived"

## Phase 2f: Memory
- 新增 `commit_memory_revision` helper（`volume_store.rs`）
- `append_to_current`：追加后 commit revision（批准文件：`current.md` + `index.md`）
- `write_volume`：封卷后 commit revision（额外含 `volumes/vol_NNN.md`）
- `asset_dir` = `session_dir`，`AssetKind::Memory`
- lazy migration

## Phase 2g: Persona
- `PersonaService::save`：在 `replace_file` 写 `personas/{pid}.json` 后 commit revision
- 直接复用 `persona.revision`（已自增）作为 `content_revision`
- 新建目录形态 `users/{uid}/personas/{pid}/` 存放 `revisions/`
- 保留 `personas/{pid}.json` 文件作为 legacy 工作副本
- `source_kind` = "manual_edit"

## 新增测试（8 个）
- `volume_store::append_to_current_creates_and_bumps_memory_revision`
- `volume_store::write_volume_creates_revision_with_volume_file`
- `domain::lorebook_write_creates_revision_dir_and_bumps_pointer`
- `domain::state_write_creates_revision_dir_and_bumps_pointer`
- `domain::persona_save_creates_revision_dir_and_bumps_pointer`
- （Phase 2c 测试由 subagent 在 domain.rs 中添加，含 character revision 验证）

## 设计决策
- 所有 service 保留 legacy 工作副本，revision 作为增量叠加
- lazy migration：旧数据无 `current_revision` 时首次写入从 revision 1 起
- State/Persona 已有 revision 计数器，直接复用（不重新从 1 起）
- Memory 的 `append_to_current` 是高频操作，每次 append 产生新 revision
- Phase 2h（trace 收口）单独 PR，依赖本 PR 的 5 个 service 先合并
