## 1. 修复正确性独立验证

### Bug 确认

`unbind_persona_endpoint` 签名（`personas.rs:193-200`）：

```rust
pub async fn unbind_persona_endpoint(
    axum::extract::State(state): axum::extract::State<Arc<DaemonState>>,
    axum::extract::Path((user_id, persona_id)): axum::extract::Path<(String, String)>,
    query: Result<axum::extract::Query<UnbindPersonaQuery>, axum::extract::rejection::QueryRejection>,
) -> Result<Json<Persona>, AirpError>
```

`UnbindPersonaQuery`（`personas.rs:258-266`）：

```rust
pub(in crate::daemon) struct UnbindPersonaQuery {
    character_id: String,           // 必填
    #[serde(default)]
    session_id: Option<String>,     // 可选
}
```

axum 的 `Query<T>` 提取器只从 URL query string 解析，不读 body。原 WebUI 代码 `client.request('DELETE', '.../bindings', { character_id: ... })` 传 JSON body，daemon 忽略 body 并因 query 缺失 `character_id` 返回 400。**Bug 确认成立。**

### 修复验证

修复后 WebUI（`console-runtime.js:243`）：

```js
client.request('DELETE', '/v1/users/.../bindings?character_id=' + encodeURIComponent(binding.control.value))
```

- 第 3 参数 `body` 未传（undefined）→ `api-client.js:152` 不发送 body ✓
- `character_id` 经 `encodeURIComponent` 编码，防特殊字符注入 ✓
- 与 Rust 集成测试 `persona.rs:468` 的 `?character_id=char-a` 调用形式一致 ✓

### POST bind 不受影响

POST bind（`console-runtime.js:236`）仍用 `client.request('POST', '.../bindings', { character_id })` 传 JSON body，对应 `BindPersonaRequest`（`personas.rs:248-256`，用 `axum::Json` 提取）。两个 handler 用不同提取器，POST 走 body、DELETE 走 query，**设计正确，互不干扰**。
